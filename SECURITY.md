# Security Policy

## Supported Versions

Only the latest major version (`v2.x.x`) currently receives security updates.
Reporting a Vulnerability.

Do **NOT** report security vulnerabilities via public GitHub issues or public discussions.

Please report security vulnerabilities by creating a private repository security advisory. This allows us to collaborate
with you on a fix privately and ensures the vulnerability is handled securely.

**Steps to report**

1. Navigate to the main page of the repository.
2. Click on the **Security** (or **Security and quality**) tab.
3. Click on **Advisories**.
4. Click **New draft advisory**.

We will acknowledge receipt within 48 hours and provide an estimated timeline for a patch. We appreciate your efforts to
responsibly disclose vulnerabilities.
